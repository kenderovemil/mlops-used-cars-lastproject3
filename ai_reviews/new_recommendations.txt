ISSUE #4: No Error Handling in Python Scripts
----------------------------------------------
SEVERITY: MEDIUM - Robustness & Debugging
LOCATIONS: All 4 Python scripts in mlops/scripts/

PROBLEM:
    Scripts lack:
    - Try-catch blocks for Azure API calls
    - Validation of environment variables before use
    - Error handling for credential acquisition
    - Proper error messages for debugging

EXAMPLE (create_compute.py):
    ```python
    subscription_id = os.environ["AZURE_SUBSCRIPTION_ID"]  # May raise KeyError
    ml_client.compute.begin_create_or_update(cpu_cluster).result()  # May fail
    ```

IMPACT:
    - Cryptic error messages when environment variables are missing
    - Difficult to debug Azure API failures
    - Workflows may hang or fail silently

FIX:
    Add comprehensive error handling:
    ```python
    import sys
    from azure.core.exceptions import AzureError
    
    try:
        # Validate environment variables
        required_vars = ["AZURE_SUBSCRIPTION_ID", "RESOURCE_GROUP", "WORKSPACE_NAME"]
        missing_vars = [var for var in required_vars if var not in os.environ]
        if missing_vars:
            raise ValueError(f"Missing required environment variables: {missing_vars}")
        
        # Get credentials
        credential = DefaultAzureCredential()
        
        # Create ML client
        subscription_id = os.environ["AZURE_SUBSCRIPTION_ID"]
        resource_group = os.environ["RESOURCE_GROUP"]
        workspace_name = os.environ["WORKSPACE_NAME"]
        
        ml_client = MLClient(
            credential=credential,
            subscription_id=subscription_id,
            resource_group_name=resource_group,
            workspace_name=workspace_name
        )
        
        # Perform operation
        result = ml_client.compute.begin_create_or_update(cpu_cluster).result()
        print(f"‚úÖ Compute cluster created/updated: {result.name}")
        
    except KeyError as e:
        print(f"‚ùå Error: Missing environment variable: {e}", file=sys.stderr)
        sys.exit(1)
    except AzureError as e:
        print(f"‚ùå Azure API Error: {e}", file=sys.stderr)
        sys.exit(2)
    except Exception as e:
        print(f"‚ùå Unexpected Error: {e}", file=sys.stderr)
        sys.exit(3)
    ```


ISSUE #5: AZURE_STORAGE_KEY Not Masked in train.yml
----------------------------------------------------
SEVERITY: HIGH - Security Risk
LOCATION: .github/workflows/train.yml, lines 36-37

PROBLEM:
    ```yaml
    - name: Set storage key
      run: |
        echo "AZURE_STORAGE_KEY=${{ secrets.AZURE_STORAGE_KEY }}" >> $GITHUB_ENV
    ```
    
    The storage key is set but NOT masked like other secrets.

IMPACT:
    - Storage key may appear in GitHub Actions logs
    - Security vulnerability if logs are accessed

FIX:
    Add masking before setting:
    ```yaml
    - name: Set storage key
      run: |
        STORAGE_KEY="${{ secrets.AZURE_STORAGE_KEY }}"
        echo "::add-mask::$STORAGE_KEY"
        echo "AZURE_STORAGE_KEY=$STORAGE_KEY" >> $GITHUB_ENV
    ```

6. ERROR HANDLING STANDARDS
   -------------------------
   All scripts must:
   - Validate environment variables at startup
   - Use try-catch for all external calls
   - Return specific exit codes (0=success, 1=config error, 2=Azure error, 3=other)
   - Log detailed error messages
   - Include troubleshooting hints in error messages
   
   Template:
   ```python
   def main():
       try:
           validate_environment()
           credential = get_credential()
           client = create_client(credential)
           result = perform_operation(client)
           print_success(result)
           return 0
       except ConfigError as e:
           print_error("Configuration", e)
           return 1
       except AzureError as e:
           print_error("Azure API", e)
           return 2
       except Exception as e:
           print_error("Unexpected", e)
           return 3
   ```
